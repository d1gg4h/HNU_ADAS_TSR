Practical example of machine learning with Iris flower data set¶

The Iris flower data set¶

The Iris flower data set is a common test case for machine learning. This data set consists of 50 samples of tree different species of Iris ( = 150 peaces of data). Each sample is defined by 4 different features. The fact, that you can define the species of the Iris by the combination of the features makes it a good case for our first machine learning approach.
Lets take a closer look at the data set.

Multilayer perceptron¶

We learned in the video(s) to neural networks that perceptrons and multilayer perceptrons are learning trough the adaption of weights during the process. The same is happening in Keras with the "Dense layer" and is described with the following formula:
output = activation(dot(input, kernel) + bias)

  • activation is the activation function
  • dot is the dot product (german: Skalarprodukt) , e.g. product of vectors
  • kernel is a weights matrix created by the layer
    Now lets combine this formula with the model of a perceptron:
    Source: own illustration

As we can see the Keras Dense layer is quite the same thing as a perceptron. Lets take a closer look at the elements of the Dense layer:

tf.keras.layers.Dense( units, activation=None, use_bias=True, kernel_initializer="glorot_uniform", bias_initializer="zeros", kernel_regularizer=None, bias_regularizer=None, activity_regularizer=None, kernel_constraint=None, bias_constraint=None, **kwargs )

Dont worry, not all elements are important for now, but we now know most of them. The most important for now are:

  • units defines our output, e.g. how our output is shaped
  • activation is the activation function we want to use
    more on activation functions

And for the first layer we need to define the shape of the input

  • input_shape=(n, m)

How do we choose the correct size of the Dense layers?¶

Unfortunately there are no exact rules, and the size needs to be determined by try and error. But there is a rule of thumb where to start:

  • First layer: Number of classes times 8
  • Second layer: Number of classes times 4
  • Third layer: Number of classes times 2
  • Forth layer: Number of classes

Compiling the model¶

Loss funciton¶

The loss function (loss) computes the difference between the actual output and the predicted output, e.g the error. There are many different loss functions for different needs.
In our case we have one category for each data point, e.g. an example can belong to one class only.
The proper loss function in this case is the categorical_crossentropy

Optimizer¶

The optimizer is updating the weight parameters (steering our model towards a right solution) to minimizing the loss function (-> minimizing the error)
Out of many optimizers we choose adam for its computational efficiency.
